本 skill 指导创作符合中文技术文档规范的内容，基于业界公认的中文技术写作标准。

用户提供需要撰写或审阅的中文技术内容：文档、博客、README、产品说明、API 参考等。

## 写作前的判断

撰写或审阅前，先明确：

- **内容类型**：教程、参考文档、概念说明、操作指南？
- **目标读者**：开发者、运维、普通用户？
- **中英文混排情况**：是否有大量英文术语、代码、命令？

## 一、标题规范

标题分为四级，使用 Markdown `#` `##` `###` `####` 表示。

### 层级原则

**（1）一级标题下不能直接出现三级标题，必须有二级标题过渡。**

```markdown
# 错误示例

### 三级标题
```

```markdown
# 正确示例

## 二级标题

### 三级标题
```

**（2）同级标题不能只有一个（孤立编号），应合并到上级或省略该层级。**

```markdown
# 错误示例

## 二级标题 A

### 三级标题 A ← 孤立，唯一的三级标题

## 二级标题 B
```

```markdown
# 正确示例

## 二级标题 A ← 直接在此处展开内容，省略孤立的三级标题

## 二级标题 B
```

**（3）下级标题不能与上级标题同名。**

```markdown
# 错误示例

## 概述

### 概述 ← 与上级标题重名
```

```markdown
# 正确示例

## 概述

### 背景 ← 使用不同名称
```

**（4）谨慎使用四级标题；三级标题下的并列内容，优先用列表代替四级标题。**

```markdown
# 结构一（不推荐，适合篇幅较长的内容）

### 三级标题

#### 四级标题 A

#### 四级标题 B

#### 四级标题 C
```

```markdown
# 结构二（推荐）

### 三级标题

**（1）A**

**（2）B**

**（3）C**
```

## 二、文本与字间距

### 中英文混排空格规则

- 全角中文字符与半角英文字符之间加一个半角空格
- 全角中文字符与半角数字之间可加可不加空格，但**全文必须统一风格**
- 英文数字与英文单位之间保留适当空隙
- 半角英文/数字与全角标点之间**不加**空格

```
错误：本文介绍如何快速启动Windows系统。
正确：本文介绍如何快速启动 Windows 系统。

错误：他的电脑是 MacBook Air 。
正确：他的电脑是 MacBook Air。

正确：今年我国经济增长率是6.5%。
正确：今年我国经济增长率是 6.5%。

正确：1 h = 60 min = 3,600 s
正确：一部容量为 16 GB 的智能手机
```

### 句子长度控制

- 单句（不含标点）控制在 20 字以内为佳；20–29 字可接受；30–39 字需语义绝对清晰；40 字以上一律不可接受
- 逗号分隔的长句，总长度不超过 100 字或正文 3 行
- 优先使用简单句和并列句，避免复合句

```
错误：本产品适用于从由一台服务器进行动作控制的单一节点结构到由多台服务器进行动作控制的并行处理程序结构等多种体系结构。

正确：本产品适用于多种体系结构。无论是由一台服务器（单一节点结构），还是由多台服务器（并行处理结构）进行动作控制，均可以使用本产品。

错误（复合句）：那个昨天生病的人没有参加会议。
正确（并列句）：他昨天生病了，没有参加会议。
```

### 写作风格

**主动语态**

```
错误：假如此软件尚未被安装，
正确：假如尚未安装这个软件，
```

**肯定句**

```
错误：请确认没有接通装置的电源。
正确：请确认装置的电源已关闭。
```

**避免双重否定**

```
错误：没有删除权限的用户，不能删除此文件。
正确：用户必须拥有删除权限，才能删除此文件。
```

**不使用非正式语言**

```
错误：Lady Gaga 的演唱会真是酷毙了，从没看过这么给力的表演！！！
正确：无法参加本次活动，我深感遗憾。
```

**不使用文言词汇**

```
错误：这是唯二的快速启动的方法。
正确：这是仅有的两种快速启动的方法。
```

**"的/地/得"用法**

```
她露出了开心的笑容。     → 形容词＋的＋名词
她开心地笑了。           → 副词＋地＋动词
她笑得很开心。           → 动词＋得＋副词
```

**代词指代必须明确**

```
错误：从管理系统可以监视中继系统和受其直接控制的分配系统。
正确：从管理系统可以监视两个系统：中继系统和受中继系统直接控制的分配系统。
```

**名词前形容词不超过两层**

```
错误：此设备的使用必须在接受过本公司举办的正式的设备培训的技师的指导下进行。
正确：此设备必须在技师的指导下使用，且指导技师必须接受过由本公司举办的正式设备培训。
```

### 英文词汇处理

**第一次出现的缩写，括号内给出中文全称**

```
IOC（International Olympic Committee，国际奥林匹克委员会）。这样定义后，便可以直接使用"IOC"了。
```

**英文原文复数翻译为中文时还原为单数**

```
英文：...information stored in random access memory (RAMs)...
中文：⋯⋯存储在随机存取存储器（RAM）里的信息⋯⋯
```

**中文正文中的英文省略号改为中文省略号**

```
英文：5 minutes later...
中文：5 分钟过去了⋯⋯
```

**英文书名/电影名改用中文表达时，双引号改为书名号**

```
英文：He published an article entitled "The Future of the Aviation".
中文：他发表了一篇名为《航空业的未来》的文章。
```

**专有名词每词首字母大写；非专有名词不大写**

```
专有名词：American Association of Physicists in Medicine（美国医学物理学家协会）→ 需大写
非专有名词：online transaction processing（在线事务处理）→ 不应大写
```

## 三、段落规范

- 每段只有一个主题；中心句放段首，后续句子为中心句服务
- 段落不超过七行，最佳四行以内
- 使用陈述和肯定语气
- 段落之间用一个空行隔开
- 段落开头不留空白字符

### 引用规范

引用第三方内容时，注明作者和出处：

```
One man's constant is another man's variable. — Alan Perlis
```

全文转载时，在开头显著位置注明作者和出处，并链接至原文：

```
本文转载自 WikiQuote
```

使用外部图片时，在图片下方或文末标明来源：

```
本文部分图片来自 Wikipedia
```

## 四、数值规范

### 半角数字

阿拉伯数字一律使用**半角**形式，不得使用全角。

```
错误：这件商品的价格是１０００元。
正确：这件商品的价格是 1000 元。
```

### 千分号

数值为千位以上应添加千分号（半角逗号）。4 位数字千分号可选；4 位以上必须加。

```
XXX 公司的实收资本为 ￥1,258,000 人民币。
```

### 货币

货币符号写在数字前，或在数字后写货币中文名称：

```
$1,000
1,000 美元
```

### 数值范围

用波浪线（`～`）或一字线（`—`）连接；带单位或百分号时两端都加单位：

```
132 kg～234 kg
67%～89%
2009 年～2011 年
```

### 增减表达

**"了"表示增量，"到"表示定量。**

```
增加到过去的两倍    → 过去为一，现在为二
增加了两倍          → 过去为一，现在为三

降低到百分之八十    → 定额是一百，现在是八十
降低了百分之八十    → 原来是一百，现在是二十
```

**不能用"降低 N 倍"或"减少 N 倍"**，改用"降低百分之几"。因为"减少一倍"逻辑上意味着数值变为零。

## 五、标点符号规范

**基本原则**

- 中文语句全部使用**全角**标点
- 整句为英文时使用**半角**标点
- 句号、问号、叹号、逗号、顿号、分号、冒号**不得出现在行首**
- 句号等点号**不得出现在标题末尾**（标号如引号、括号、问号可以）

### 句号

中文语句结尾用全角句号（`。`）。括号注释时，句号在括号**外**：

```
错误：关于文件的输出，请参照第 1.3 节（见第 26 页。）
正确：关于文件的输出，请参照第 1.3 节（见第 26 页）。
```

### 逗号

逗号（`，`）表示句子内部的一般性停顿。注意避免"一逗到底"——整段除结尾外全部用逗号，应适当改用分号或分句。

### 顿号

句子内部的并列词（含英文词）用全角顿号（`、`）分隔，不用逗号：

```
错误：我最欣赏的科技公司有 Google, Facebook, 腾讯, 阿里和百度等。
正确：我最欣赏的科技公司有 Google、Facebook、腾讯、阿里和百度等。
```

最后一项用"和"连接，比只用顿号更连贯：

```
可以：我最欣赏的科技公司有 Google、Facebook、腾讯、阿里，以及百度等。
更优：我最欣赏的科技公司有 Google、Facebook、腾讯、阿里和百度等。
```

英文句子中并列词使用半角逗号：

```
Microsoft Office includes Word, Excel, PowerPoint, Outlook and other components.
```

### 引号

引用时使用全角双引号（`" "`），注意前后引号不同。引号里再用引号时，外层双引号，内层单引号（`' '`）：

```
许多人都认为客户服务的核心是"友好"和"专业"。

鲍勃解释道："我要放音乐，可萨利说，'不行！'。"
```

### 括号

补充说明时使用全角圆括号（`（）`），括号前后不加空格：

```
请确认所有的连接（电缆和接插件）均安装牢固。
```

### 冒号

全角冒号（`：`）引出解释和说明：

```
请确认以下几项内容：时间、地点、活动名称和来宾数量。
```

表示时间时使用半角冒号（`:`）：

```
早上 8:00
```

### 省略号

省略号（`⋯⋯`）占两个汉字空间、包含六个省略点。不得使用 `...` 或 `。。。`。省略号不与"等"连用：

```
错误：我们为会餐准备了香蕉、苹果、梨…等各色水果。
正确：我们为会餐准备了各色水果，有香蕉、苹果、梨⋯⋯
正确：我们为会餐准备了香蕉、苹果、梨等各色水果。
```

### 感叹号

尽量使用平静语气，避免感叹号（`！`）。不得多个连用：

```
错误：这个功能真是太好用了！！！
正确：这个功能易于使用。
```

### 破折号

破折号一般用于进一步解释，应占两个汉字的位置。两种写法均可：

```
直觉————尽管它并不总是可靠的————告诉我，这事可能出了些问题。

直觉 —— 尽管它并不总是可靠的 —— 告诉我，这事可能出了些问题。
```

### 连接号

两个名词的复合，或图表编号，使用直线连接号（`-`，半角）：

```
氧化-还原反应
图 1-1
```

数值范围使用波浪连接号（`～`）或一字号（`—`），占一个全角字符，注意两端加单位：

```
2009 年～2011 年
周围温度：-20 °C 至 -10 °C
```

## 六、文档体系

### 软件文档推荐结构

| 章节                        | 必要性 | 说明                                       |
| --------------------------- | ------ | ------------------------------------------ |
| 简介（Introduction）        | 必备   | 产品和文档的总体说明                       |
| 快速上手（Getting Started） | 可选   | 最快速使用路径                             |
| 入门篇（Basics）            | 必备   | 环境准备、安装、配置                       |
| 进阶篇（Advanced）          | 可选   | 中高级开发教程                             |
| API 参考（Reference）       | 可选   | API 逐一说明                               |
| FAQ                         | 可选   | 常见问题                                   |
| 附录（Appendix）            | 可选   | 术语表、最佳实践、故障排查、更新日志、反馈 |

参考范例：[Redux 手册](https://redux.js.org/introduction/getting-started)、[Atom 手册](http://flight-manual.atom.io/)

### 文件命名规范

- 文件名不含空格，只使用半角字符，不使用中文
- 只使用小写字母（README、LICENSE 等说明文件除外）
- 多单词用半角连词线（`-`）分隔，不用下划线

```
错误：名词解释.md
正确：glossary.md

错误：TroubleShooting.md
正确：troubleshooting.md

错误：advanced_usage.md
正确：advanced-usage.md
```

## 审阅清单

审阅中文技术文档时，逐项检查：

- [ ] 中英文之间有半角空格，且全文风格统一
- [ ] 数字使用半角，千位以上有千分号
- [ ] 标点全角/半角使用正确
- [ ] 省略号使用 `⋯⋯`，未与"等"连用
- [ ] 无双重否定、无被动语态
- [ ] 单句不超过 40 字
- [ ] 段落不超过 7 行，首句为中心句
- [ ] 标题层级完整，无孤立编号，无重名，无跳级
- [ ] 英文缩写首次出现时有括号注释
- [ ] 文件名全小写、用连词线分隔、无中文
